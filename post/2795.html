1. 变量说明

1.1 指定生成文件系统镜像类型

IMAGE_FSTYPES += "tar.gz tar.bz2"

1.2 指定镜像文件的基本名称

该变量默认为菜谱名称 ${PN}

export IMAGE_BASENAME = "arago-base-tisdk-image"

1.3 指定需要安装哪些包到镜像中

IMAGE_INSTALL += "\
    packagegroup-arago-base \
    php \
    iperf \
    "

1.4 指定配方的许可证

LICENSE = "MIT"

1.5 设置目标镜像的特性

IMAGE_FEATURES += "package-management splash"

不止是包和包组，这些特性来自于 IMAGE_FEATURES 的定义，假设 :

FEATURE_PACKAGES_widget = "package1 package2"

那么，如果 IMAGE_FEATURES 中包含了 widget ，就包含了 package1 和 package2 。例如：

MACHINE=zc706-zynq7 bitbake -e petalinux-image | grep ^FEATURE_PACKAGES

FEATURE_PACKAGES_ssh-server-dropbear="packagegroup-core-ssh-dropbear"
FEATURE_PACKAGES_petalinux-benchmarks="packagegroup-petalinux-benchmarks"
FEATURE_PACKAGES_petalinux-base="packagegroup-petalinux"
FEATURE_PACKAGES_x11="packagegroup-core-x11"
FEATURE_PACKAGES_nfs-server="packagegroup-core-nfs-server"
FEATURE_PACKAGES_eclipse-debug="packagegroup-core-eclipse-debug"
FEATURE_PACKAGES_petalinux-qt="packagegroup-petalinux-qt"
FEATURE_PACKAGES_x11-base="packagegroup-core-x11-base"
FEATURE_PACKAGES_petalinux-audio="packagegroup-petalinux-audio"
FEATURE_PACKAGES_petalinux-gstreamer="packagegroup-petalinux-gstreamer"

IMAGE_FEATURES 中定义的petalinux-qt 就表示 packagegroup-petalinux-qt ，可以在sources目录中直接搜索这个包组的 .bb 文件

1.6 定位配方文件

BBPATH .= ":${LAYERDIR}"

用来定位.bb、.bbclass和配置文件等

1.7 层路径变量(LAYERDIR)

BBPATH .= ":${LAYERDIR}"

当在layer.conf配置文件中使用时，该变量提供当前层的路径

1.8 指定新建的layer和openembedded-core版本兼容

LAYERSERIES_COMPAT_layer_root_name = "dunfell zeus"

LAYERSERIES_COMPAT_meta-zholpw01-4376 = "thud"

指定新创建的层和openbedded - core层哪个版本是兼容的，如果没有设置该变量，OpenEmbedded 构建系统就会生成一个警告，如下图

1.9 列出已配置层的名称，用于查找其他BBFILE_*变量

BBFILE_COLLECTIONS += "meta-ti"

BBFILE_COLLECTIONS是层名字列表

1.10 匹配特定层的BBFILES变量

BBFILE_PATTERN_meta-ti := "^${LAYERDIR}/"

1.11 指定层优先级

BBFILE_PRIORITY_meta-ti = "6"
  • 通过设置这个变量，您可以将一个层与包含相同配方的其他层区分优先级，从而有效地控制多个层的优先级。通过这个变量建立的优先级与菜谱的版本无关(PV变量)。例如，如果一层的配方具有较高的PV值，但其BBFILE_PRIORITY被设置为具有较低的优先级，那么该层仍然具有较低的优先级。
  • BBFILE_PRIORITY变量的值越大，优先级越高。例如，6的优先级高于5。
  • 可以使用bitbake-layers show-layers命令可以列出所有已配置的层及其优先级。

1.12 指定许可证路径

LICENSE_PATH += "${LAYERDIR}/licenses"

构建期间使用的附加许可的路径。默认情况下，OpenEmbedded构建系统使用COMMON_LICENSE_DIR来定义包含构建过程中使用的公共许可文本的目录。LICENSE_PATH变量允许您将该位置扩展到其他具有附加许可的区域

1.13 指定需要依赖的层

LAYERDEPENDS_meta-arago-distro = " \
    core \
    meta-ti \
    openembedded-layer \
    networking-layer \
    meta-python \
    linaro-toolchain \
    meta-optee \
    qt5-layer \
    browser-layer \
    meta-arago-extras \
"

1.14 指定层的版本号

LAYERVERSION_meta-ti = "1"

1.15 BB_DANGLINGAPPENDS_WARNONLY

BB_DANGLINGAPPENDS_WARNONLY = "1"   #出现警告，不退出

BB_DANGLINGAPPENDS_WARNONLY = "0"   #出现错误，退出

定义BitBake如何处理附加文件(.bbappend)没有对应的配方文件(.bb)的情况。

1.16 RRECOMMENDS

RRECOMMENDS_${PN} = "depth-image-proc image-proc nodelet tf"

扩展正在构建的包的可用性的包列表。正在构建的包并不依赖于这个包列表来成功构建，而是使用它们来扩展可用性。

1.17 BAD_RECOMMENDATIONS

BAD_RECOMMENDATIONS += " \
    ti-llvm3.6-dev \
    opencl-monitor-dev \
    libulm-dev \
    gdbserver-c6x-dev \
    coreutils \
    coreutils-dev \
"

可以通过在BAD_RECOMMENDATIONS变量中列出RRECOMMENDS指定的包来防止它们被安装

1.18 ROOTFS_POSTPROCESS_COMMAND

create_modules_dir() {
    if [ ! -d ${IMAGE_ROOTFS}${base_libdir}/modules ]; then
        mkdir ${IMAGE_ROOTFS}${base_libdir}/modules
    fi
}

ROOTFS_POSTPROCESS_COMMAND += "create_modules_dir;"

指定OpenEmbedded构建系统创建根文件系统后要调用的函数列表。可以指定用分号分隔函数
