gdb使用

​ gdb 工具是 GNU 项目调试器，基于命令行。和其他的调试器一样，我们可以使用 gdb 来一行行的运行程序、单步执行、跳入/跳出函数、设置断点、查看变量等等，它是 UNIX/LINUX 操作系统下强大的程序调试工具。 gdb 支持多种语言，包括 Ada、汇编、 C/C++、 D、 Fortran、 GO、Objective-C、 OpenCL、 Modula-2、 Pascal 和 Rust。

gdb官网：https://www.gnu.org/software/gdb/

1. gdb基本命令

  • 测试代码

    #include <stdio.h>

void show()
{
    printf("show\n");
}

int main(int argc, char *argv[])
{
    int arr[4] = {1, 2, 3, 4};
    int i = 0;

    for (i = 0; i < 4; i++) {
        printf("arr[%d]: %d\n", i, arr[i]);
    }

    show();

    return 0;
}

  • 编译

    gcc **-g** main.c

    编译的时候必须加上 "-g" 选项，这样编译出来的可执行文件才带有调试信息

  • 使用gdb调试

    gdb a.out

1.2 run命令

  • 用于启动程序，简写：r

1.3 quit命令

  • 退出gdb模式，简写：q

1.4 break命令

  • 设置断点，简写：b

1.5 info命令

  • 用于显示有关正在调试的程序的内容的通用命令

1.5.1 info breakpoints(b)

  • 显示断点的状态

1.6 list命令

  • ​ 用于列出源码，简写：l

1.7 print命令

  • 用于打印变量或者变量地址，简写：p

1.8 step命令

  • 单步调试，此命令会进入到函数里面调试

1.9 next命令

  • 单步调试，此命令不会进入到函数里面调试

1.10 shell命令

  • 用于执行shell命令

1.10 watchpoint命令

  • 用于观察变量是否变化

2. gdb技巧

2.1 日志功能

  • 用于记录在gdb调试的日志，执行以下指令开启日志功能

    set logging on
